overview

Searching and sorting are two fundamental parts of computer science. Searching is the process of finding a particular item or element in a collection of data, and sorting is the process of arranging the elements in a particular order, such as ascending or descending order.

searching (java)

what is it?

Searching, or searching algorithms, are used to find an element within any data structure, most commonly, arrays. Searching algorithms are made to make the process faster and more time and memory efficient.

how? + examples

There are multiple ways to search in Java. Two searching algorithms that will be covered in this tutorial are sequential search and binary search.

Sequential Search

The naive way to search is sequential search. Sequential search loops through and checks every element in the array whether or not it‚Äôs equal to the desired element.

int search(int arr[], int x){
              int N = arr.length;
              for (int i = 0; i < N; i++) {
                  if (arr[i] == x)
                      return i;
              }
              return -1;
          }

This has a time complexity of O(N)O(N)O(N)﻿ and space complexity of O(1)O(1)O(1)﻿. This is not that great.

Binary Search

Binary Search is a more efficient and more commonly used way to search. It can be done iteratively and recursively where the array to be searched from must be sorted. The way this algorithm works is that it gets the middle element of the array and compares whether it is greater, lower, or equal to the target, then taking the middle element of the half the target is in and comparing with its middle number as well. This continues until the element is found, or is narrowed down to a subarray of length 1 (where lowest and highest number of the subarray are equal). 

int search(int arr[], int x){
              int low = 0, high = arr.length - 1;
              while (low <= high) {
                  int mid = low + (high - low) / 2;
                  if (arr[mid] == x)
                      return mid;
                  if (arr[mid] < x)
                      low = mid + 1;
                  else
                      high = mid - 1;
              }
              return -1; //not found
          }

This has a time complexity of O(log‚Å°N)O(\log N)O(logN)﻿ and space complexity of O(1)O(1)O(1)﻿. This is pretty good.

common mistakes

Sequential Search

  • using it for large datasets as it will be slow

Binary Search

  • not sorting the array
  • mixing up the conditions and indices ‚Äî ie. low = mid+1 (correct) vs low = mid vs low = mid-1
  • integer overflow when calculating for mid ‚Äî int mid = low + (high-low)/2 would bypass any potential integer overflowing

sorting

what is it?

Sorting, or sorting algorithms, are used to rearrange a data structure, most commonly an arrays according to comparison operators which will decide precedence and the general order.

how? + examples

There are multiple ways to sort data. Two sorting algorithms we will discuss are bubble sort and radix sort.

Bubble Sort

Basically you iterate through the entire array and swap elements that are not in the right precedence. That repeats until it‚Äôs in order.

void sort(int arr[]){
              int N = arr.length;
              for (int i = 0; i < N - 1; i++)
                  for (int j = 0; j < N - i - 1; j++)
                      if (arr[j] > arr[j + 1]) {
                          int temp = arr[j];
                          arr[j] = arr[j + 1];
                          arr[j + 1] = temp;
                      }
          }

This has a time complexity of O(N2)O(N^2)O(N2)﻿ and a space complexity of O(1)O(1)O(1)﻿.
