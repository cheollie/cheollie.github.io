SQL Injection

used when the site uses SQL, a language used to obtain information from databases

when input is not sanitized when placed into a SQL query, it can cause unintended and malicious behavior

one of the most common and basic examples is login without password

SELECT * FROM users WHERE username='$username' AND password = '$password'

$username and $password are variables for whatever is inputted by the user

so if i input username as coolperson and password as 123456, the query is

SELECT * FROM users WHERE username='coolperson' AND password = '123456'

but what if you put characters that would be confused as part of the command in your input? what if username was coolperson'-- (--, /* comments out the rest of the line in SQL) the command would be

SELECT * FROM users WHERE username='coolperson'--' AND password = '123456'

you can see the rest of the command is ignored because of the comment and the quotation mark at the end of our input was interpreted as the end of the string â€” so this would login if coolperson was a user in the database

but what if we don't know what user is in the database â€” using knowledge gained from here and other programming knowledge, what inputs do you think would return a result despite not knowing any usernames or passwords?

  • answer 
    SELECT * FROM users WHERE username='' OR 1=1 --' AND password = '123456'

    the ' closes off the username string, OR will return true if either username='' or 1=1 is true, and since 1=1 is always true, it will return true; -- comments the rest of the command out

using SQL injection, we can force databases to return data not meant to be returned!

As this is the basics, further manipulation can be done with other keywords/operations such as:

  • || string concatination
  • UNION
  • LIKE
